1. 共病用药安全性筛查（逻辑交集）
自然语言描述：查找治疗 [疾病A] 但对 [疾病B] 禁用的药物。
Cypher 模版：
cypher
MATCH (dis1:Disease {name: "{disease_A}"})<-[:indication]-(d:Drug)-[:contraindication]->(dis2:Disease {name: "{disease_B}"})
RETURN d.name AS DrugName

示例：
{disease_A} = "Hypertension" (高血压)
{disease_B} = "Asthma" (哮喘)
2. 基于特定转运体/酶的药物机制查找
自然语言描述：查找明确使用 [蛋白名称] 作为转运体（Transporter）的药物。
Cypher 模版：
cypher
MATCH (d:Drug)-[:transporter]->(p:Protein {name: "{protein_name}"})
RETURN d.name AS DrugName

示例：
{protein_name} = "ABCB1"
3. 药物副作用的解剖学定位（三跳路径）
自然语言描述：查找 [药物名称] 的靶点蛋白主要在哪些解剖部位有表达。
Cypher 模版：
cypher
MATCH (d:Drug {name: "{drug_name}"})-[:target]->(p:Protein)-[:expression present]->(a:Anatomy)
RETURN DISTINCT p.name AS TargetProtein, a.name AS AnatomicalRegion

示例：
{drug_name} = "Aspirin" (阿司匹林)
4. 基于靶点的药物-通路机制探索（三跳路径）
自然语言描述：查找 [药物名称] 通过其靶点蛋白影响了哪些生物通路。
Cypher 模版：
cypher
MATCH (d:Drug {name: "{drug_name}"})-[:target]->(p:Protein)-[:pathway_protein]-(path:Pathway)
RETURN DISTINCT p.name AS Target, path.name AS Pathway

示例：
{drug_name} = "Metformin" (二甲双胍)
5. 疾病间的表型重叠（鉴别诊断）
自然语言描述：查找 [疾病A] 和 [疾病B] 之间共有的表型（症状）。
Cypher 模版：
cypher
MATCH (dis1:Disease {name: "{disease_A}"})-[:phenotype present]->(ph:Phenotype)<-[:phenotype present]-(dis2:Disease {name: "{disease_B}"})
RETURN ph.name AS SharedPhenotype

示例：
{disease_A} = "Autism" (自闭症)
{disease_B} = "Fragile X syndrome" (脆性X综合征)

1. 基于表型的潜在药物关联查找（症状缓解）
自然语言描述：查找针对某些基因/蛋白的药物，而这些基因/蛋白已知会导致 [疾病名称] 所表现出的某种表型（症状）。这通常用于寻找可能缓解疾病症状的药物。
Cypher 模版：
cypher
MATCH (d:disease {name: "{disease_name}"})-[:disease_phenotype_positive]->(ph:effect_phenotype)
MATCH (g:gene_protein)-[:phenotype_protein]->(ph)
MATCH (dr:drug)-[:drug_protein]->(g)
RETURN DISTINCT dr LIMIT 50

示例：
{disease_name} = "Alzheimer's disease" (阿尔茨海默病)
2. 基于解剖部位的疾病风险探索
自然语言描述：查找在 [解剖部位] 表达的蛋白，并找出这些蛋白关联了哪些疾病。用于探索特定器官可能发生的疾病风险。
Cypher 模版：
cypher
MATCH (p:gene_protein)-[:anatomy_protein_present]->(a:anatomy {name: "{anatomy_name}"})
MATCH (p)-[:disease_protein]->(d:disease)
RETURN DISTINCT d LIMIT 50

示例：
{anatomy_name} = "Liver" (肝脏)
3. 药物作用通路查询
自然语言描述：查找 [药物名称] 通过其靶点蛋白，最终影响了哪些生物通路。这有助于理解药物的作用机理（MOA）。
Cypher 模版：
cypher
MATCH (d:drug {name: "{drug_name}"})-[:drug_protein]->(p:gene_protein)
MATCH (p)-[:pathway_protein]->(pw:pathway)
RETURN DISTINCT pw LIMIT 50

示例：
{drug_name} = "Imatinib" (伊马替尼)
4. 实体间特定三跳路径查询（通用路径发现）
自然语言描述：查找从 [起始实体] 到 [终止实体] 之间，严格遵循 [关系1] -> [关系2] -> [关系3] 模式的路径。这是一个高度灵活的模版，用于验证特定的假设路径。
Cypher 模版：
cypher
MATCH (a:{start_label} {name: "{start_name}"})
MATCH (b:{end_label} {name: "{end_name}"})
MATCH p = (a)-[:{rel_1}]-()-[:{rel_2}]-()-[:{rel_3}]-(b)
RETURN p LIMIT 20

(注：在实际代码中，Label和Relationship Type通常不能直接作为参数传递，需要使用字符串拼接方式构建查询，或者使用 APOC 库)
示例：
{start_label} = "drug", {start_name} = "Metformin"
{end_label} = "disease", {end_name} = "Diabetes Mellitus"
{rel_1} = "drug_protein", {rel_2} = "protein_pathway", {rel_3} = "pathway_disease"
5. 药物多靶点统计（多重药理学分析）
自然语言描述：统计数据库中所有药物的靶点数量，并按靶点数量从高到低排序，返回前 50 个“最广谱”的药物。这不需要输入参数，是一个全局统计查询。
Cypher 模版：
cypher
MATCH (d:drug)-[:drug_protein]->(p:gene_protein)
WITH d, COUNT(p) as cnt
ORDER BY cnt DESC
RETURN d, cnt LIMIT 50

示例：
无参数（直接运行以获取多靶点药物列表）
